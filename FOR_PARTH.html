2 How to Run the Project Locally

Option A — Plain Python (Recommended to start)

Clone the repo 
git clone https://github.com/parthanallelu/Churn_predicition.git
cd Churn_predicition
Create a virtual environment (optional but recommended) 
python -m venv venv
source venv/bin/activate       # Mac/Linux
venv\Scripts\activate             # Windows
Install dependencies 
pip install -r requirements.txt
Start the server 
python app.py
Open http://localhost:8080 in your browser.

Option B — Docker (One command)

docker-compose up --build

Open http://localhost:8080. Stop with Ctrl+C, then docker-compose down.

3 How to Run Tests

Install test runner first:

pip install pytest pytest-cov

Run all tests:

pytest tests/ -v

Run with coverage report:

pytest tests/test_model.py --cov=custom_train --cov-report=term-missing

You should see ~40 tests passing. The model unit tests (test_model.py) do not need the CSV — they use synthetic data. The API tests (test_api.py) require the .pkl files to exist.

4 What You Need to Do — Model Improvement

The only thing left that requires your action is improving the model accuracy.

Current performance: Accuracy 57.2% · Precision 37.5% · F1 0.486 — this is barely better than random. Everything else in the project is solid, but this is the main gap before it's truly world-class.

You need the cell2celltrain.csv file on your machine. Once you have it, follow these steps exactly.

Step 1 — Add feature engineering to custom_train.py

After the imputation loop (around line 303 in custom_train.py, after all raw rows are processed into X_raw), add these derived features before encoding:

# After all rows are in X_raw, before encoding
# Add 4 derived features per row to boost signal

# First extend feature_cols list
feature_cols += ['RevenuePerMinute', 'OverageRatio', 'ChargeToRevenueRatio', 'MonthsPerDay']

# Then add the calculated values to each row
monthly_min_idx   = feature_cols.index('MonthlyMinutes')
overage_min_idx   = feature_cols.index('OverageMinutes')
revenue_idx       = feature_cols.index('MonthlyRevenue')
charge_idx        = feature_cols.index('TotalRecurringCharge')
months_idx        = feature_cols.index('MonthsInService')
equip_days_idx    = feature_cols.index('CurrentEquipmentDays')

for row in X_raw:
    mm   = float(row[monthly_min_idx]) if row[monthly_min_idx] else 0
    om   = float(row[overage_min_idx]) if row[overage_min_idx] else 0
    rev  = float(row[revenue_idx])     if row[revenue_idx]     else 0
    chg  = float(row[charge_idx])      if row[charge_idx]      else 0
    mos  = float(row[months_idx])      if row[months_idx]      else 0
    eqd  = float(row[equip_days_idx])  if row[equip_days_idx]  else 1

    row.append(rev  / (mm + 1))    # revenue per minute
    row.append(om   / (mm + 1))    # overage ratio
    row.append(chg  / (rev + 1))   # charge to revenue ratio
    row.append(mos  / eqd)         # months of service per equipment day

Step 2 — Increase threshold sampling & tree count

In custom_train.py, find line ~74 (the threshold sampling limit) and change it:

# FROM:
if len(possible_thresholds) > 15:
    possible_thresholds = random.sample(list(possible_thresholds), 15)

# TO:
if len(possible_thresholds) > 25:
    possible_thresholds = random.sample(list(possible_thresholds), 25)

Also increase the number of estimators (around line 348):

# FROM:
bagging_model = CustomBaggingClassifier(n_estimators=30)

# TO:
bagging_model = CustomBaggingClassifier(n_estimators=50)

Step 3 — Try different max_depth values

In the fit method of CustomBaggingClassifier (around line 190), change:

# Try each of these one at a time, retrain, check F1 score
tree = CustomDecisionTree(max_depth=8)    # current (baseline)
tree = CustomDecisionTree(max_depth=10)   # try this
tree = CustomDecisionTree(max_depth=12)   # try this

Pick whichever depth gives the highest F1 score on the test set.

Step 4 — Retrain

python custom_train.py    # retrain with new features + hyperparams
python generate_graphs.py # regenerate EDA charts

Check the printed output. You're targeting: Accuracy > 75% and F1 > 0.70.
