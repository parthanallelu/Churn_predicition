Behind the Mathematics

Understanding how the predictive engine works without external libraries.

1. Problem Statement

In the highly competitive telecommunications sector, acquiring new customers is significantly more expensive than retaining existing ones. Companies frequently lose revenue due to "customer churn"—when subscribers abruptly cancel their services in favor of competitors.

The problem lies in accurately identifying exactly which active customers currently possess a high mathematical probability of churning before they actually leave, allowing the business to proactively intervene with targeted retention incentives.

2. Solution Approach

Rather than relying on abstracted "black-box" libraries (like Scikit-Learn or Pandas), we engineered a completely custom predictive Machine Learning platform natively in Python mapping directly to browser Glassmorphism REST interactions.

  • Native Parsing: Utilizing built-in csv loops to manually track integer bounds and identify missing strings (Mean Imputation algorithms natively).
  • Algorithmic Construction: Manually engineering recursive arrays computing mathematical variance limits.
  • Lightweight Serving: Deserializing pickle logic directly into a Flask router, creating an instant evaluation cycle responding directly to JSON payloads completely independent of data science frameworks.

3. Algorithms Used

Decision Trees (The Base Learner)

A Decision Tree functions by splitting the data sequentially via IF/ELSE structural questions based upon feature variance. Our custom CustomDecisionTree class algorithm operates recursively:

  • It analyzes a node's probability threshold spanning every single feature category.
  • Calculates the specific theoretical mathematical cleanliness (Information Gain) for every theoretical split.
  • Slices the array matrix in half based on the lowest impurity ratio and re-loops until it hits the configured max_depth node floor.

Bootstrap Aggregating (Bagging Ensemble)

Because singular Decision trees are prone to "overfitting" (memorizing standard curves identically rather than generalizing), we built a wrapper class called CustomBaggingClassifier.

This algorithm leverages the random library to execute "Sampling with Replacement." It loops through the dataset, extracting randomly sequenced rows into 10 separate completely independent datasets. It then trains 10 independent Decision trees on these random copies. When a prediction is requested, all 10 trees vote independently, and the strict majority probability ratio is returned.

4. Formulas & Mathematical Matrices

Gini Impurity (Algorithm Branching)

The core mathematical engine deciding how to construct the tree array bounds relies on probabilities:

Gini = 1 - Σ (pi)2

Where pi is the probabilistic fraction of objects that physically belong to class i within a specific mathematical split. A Gini of 0 means perfect purity.

Evaluation Metric Validations

Our custom analysis scripts process predictive accuracies recursively against native Python variable tracking loops, yielding absolute precision curves:

Accuracy = (TP + TN) / (Total Observations)

Precision = TP / (TP + FP)

Recall = TP / (TP + FN)

We completely discard reliance on metrics.accuracy_score to generate a dynamic custom_plot_data.json object which bridges the algorithmic output strictly into our browser layouts.
